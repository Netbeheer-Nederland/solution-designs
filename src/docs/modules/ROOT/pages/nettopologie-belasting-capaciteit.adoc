== Reikwijdte

Het doel is om belanghebbende (Technische Adviseurs en Derden) voldoende informatie over het net te geven waarmee ze in staat zijn om gerichte individuele of collectieve voorstellen voor GV-klanten te doen, waardoor het net beter wordt benut.

// Bron: Deepdives nettopologie capaciteit belasting_v0.2.pdf, pagina 21

.Rollen
[cols="1, 2"]
|===
|Rol |Beschrijving

|Netbeheerder
|Vennootschap die op grond van de wet is aangewezen voor het beheer van een of  meer energienetten, niet zijnde gesloten distributiesystemen.

|GV-klant
|Afnemer met een aansluiting met een totale maximale capaciteit groter dan 3x80  Ampère voor elektriciteit of 40 m^3^(n) per uur voor gas, of een aansluiting op een gesloten distributiesysteem, of een aansluiting van een afnemer zoals bedoeld in artikel 1, tweede of derde lid, van de Elektriciteitswet 1998.

|Technisch adviseur
|Een organisatie of persoon die faciliteert bij het zoeken naar oplossingen voor congestieproblematiek door energie-advies te geven.

|Derden
|Derden die vanuit overige use cases belang hebben bij de informatie verstrekt vanuit dit dataproduct.

|===

Het volgende gebruiksscenario wordt ondersteund:

----

}

"TA": "Technisch\nAdviseur"

"TA" -> "GV-klant": "1. Verzoekt om NBC tbv advies"
"GV-klant" -> "NB": "2. Verzoekt om NBC"
"NB" -> "GV-klant": "3. Verzoekt om authenticatie \nen welke EAN"
"GV-klant" -> "NB": "4. Authenticeert en levert EAN"
"NB" -> "NB": "5. Bepaalt NBC op basis van EAN\nvoor afgaand veld + station"
"NB" -> "GV-klant": "6. Levert NBC-gegevens"
"GV-klant" -> "TA": "7. Levert NBC-gegevens"
"TA" -> "TA": "8. Maakt advies"
"TA" -> "GV-klant": "9. Geef advies"

----

1. De Technisch Adviseur verzoekt de GV-klant om NBC-gegevens die hij nodig heeft voor zijn advies;
2. De GV-klant vraagt zijn netbeheerder om NBC-gegevens van een specifiek gebied;
3. De netbeheerder authenticeert de GV-klant en verzoekt om de betreffende EAN;
4. De GV-klant authenticeert zich bij de netbeheerder en levert de betreffende EAN;
5. De netbeheerder bepaalt het afgaande veld en het bijbehorende station horende bij dit EAN en vervolgens de NBC-gegevens van een specifiek gebied behorende bij dit station;
6. De netbeheerder levert de gevraagde NBC-gegevens aan de GV-klant;
7. De GV-klant levert de NBC-gegevens aan de Technische Adviseur; 
8. De Technische Adviseur gebruikt de NBC-gegevens om een advies te maken voor de EAN van de GV-klant;
9. De Technische Adviseur geeft zijn advies aan de GV-klant.

== Dataset

De dataset beschrijft van een EAN de nettopologie, de belasting en de capaciteit (operationele limiet) van het afgaande veld dat bij deze EAN hoort en haar transformator.

== Voorwaarden
Dit dataproduct wordt als _gesloten data_ aangeboden, onder de _Grondslag Toestemming_. Voor een GV-klant is toestemming impliciet verleend voor eigen aansluiting(en) vanuit de authenticatie en autorisatie van de GV-klant door de netbeheerder.

// == Technische en ambigue begrippen
// 
// In dit ontwerp worden een aantal technische of ambigue begrippen gebruikt. Hun formele definitie staat in het begrippenmodel. Vaak is zo'n definitie echter te complex of te abstract voor spreektaal. Daarom volgt hieronder een informele definitie (interpretatie) in spreektaal.
// 
// [cols="2, 8"]
// |===
// |Begrip |Informele definitie (interpretatie)
// 
// |Aansluiting |Recht op gebruik van een verbinding
// |Belasting |Een waarde die aangeeft welk deel van de capaciteit benut wordt
// |Capaciteit |Een waarde die aangeeft wat een elektrische geleider zou kunnen
// |Congestie |Een afname of invoeding die de capaciteit van een elektrische geleider overtreft
// |Energiehub |Een samenwerking tussen bedrijven en organisaties om hun lokale energievoorziening te organiseren
// |Handelingsperspectief |Het hebben van mogelijkheden om in moeilijke situaties toch te kunnen handelen 
// |Nettopologie |Het netwerk op een gegeven moment in de tijd
// |Operationele limiet |De hoogste waarde die tijdens de uitvoering toegestaan kan worden. Verwant aan zekeringswaarde, maar niet altijd daaraan gelijk
// |Overdrachtspunt |De overeengekomen grens waarop de verantwoordelijkheid overgaat van het energienet van de netbeheerder naar de installatie van de aangeslotene, of tussen twee energienetten
// |Restcapaciteit |Dat deel van capaciteit dat niet benut wordt. Verwant aan vrije ruimte, maar niet altijd daaraan gelijk 
// |Verbinding |Object waar energie door getransporteerd wordt
// |Vrije ruimte |Dat deel van capaciteit dat niet benut wordt. Verwant aan restcapaciteit, maar niet altijd daaraan gelijk 
// |Zekeringswaarde |Waarde die aangeeft hoe zwaar een zekering moet zijn om een elektrisch circuit te beschermen tegen overbelasting en kortsluiting. Verwant aan operationele limiet, maar niet altijd daaraan gelijk
// 
// |===
